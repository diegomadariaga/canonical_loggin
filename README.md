## Description

A NestJS application demonstrating the **Canonical Logging** pattern using Pino (`nestjs-pino`) and AsyncLocalStorage (`nestjs-cls`).

This pattern consolidates request metadata (HTTP method, path, status, latency, request/response size) and custom app metrics (DB query counts, external API durations) into **exactly one** structured log line at the end of each request lifecycle.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Canonical Logging Demo (Testing)

Run the server on a port of your choice (e.g., `3001`):
```bash
$ PORT=3001 npm run start
```

Use the following curl commands in another terminal to test the endpoints and verify the single canonical log line produced for each:

### 1. Simple successful request (1 DB query)
```bash
curl -i http://localhost:3001/
```

### 2. Complex successful request (3 DB queries, 2 External APIs, Custom User ID)
```bash
curl -i http://localhost:3001/success -H "x-user-id: usr_custom_123"
```

### 3. Slow request (250ms latency + 1 DB query)
```bash
curl -i "http://localhost:3001/slow?delay=250"
```

### 4. Client validation error (400 Bad Request)
```bash
curl -i "http://localhost:3001/error?type=bad_request"
```

### 5. Server crash error (500 Internal Server Error)
```bash
curl -i http://localhost:3001/error
```

### 6. JSON Production Mode logging
Toggle JSON output formatting for production log parsers:
```bash
$ NODE_ENV=production PORT=3001 npm run start
```
